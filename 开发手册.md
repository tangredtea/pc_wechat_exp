## 项目架构

```
wechat_exp/
├── README.md                   # 用户文档
├── 开发手册.md                  # 本文档
├── build.bat                   # PyInstaller 一键打包脚本
├── wechat_exp.spec             # PyInstaller spec 文件
├── requirements.txt            # Python 依赖
├── src/                        # 源码
│   ├── main.py                 # 入口：交互菜单 + 命令行模式
│   ├── key_scan.py             # Win32 内存扫描密钥提取
│   ├── decrypt.py              # SQLCipher 4 批量解密
│   ├── chat_list.py            # 聊天列表扫描 + 群名/人名解析
│   ├── chat_export.py          # 聊天记录导出引擎
│   ├── employee_match.py       # 员工 Excel 匹配引擎
│   ├── wordcloud_gen.py        # 词云分析 + ECharts HTML
│   ├── report_gen.py           # HTML 综合报告
│   ├── utils.py                # 公共工具函数
│   ├── stopwords.py            # 智能停用词系统
│   └── constants.py            # 消息类型、时区、颜色常量
└── output/                     # 运行时自动生成（可删除重建）
    ├── all_keys.json           # 提取的密钥
    ├── decrypted/              # 解密后的数据库（保持原始目录结构）
    ├── exports/                # 聊天记录 TXT
    ├── employee_export/        # 员工批量导出 TXT
    ├── wordcloud/              # 词云 HTML
    └── report.html             # 综合报告
```

### 数据流

```
Weixin.exe 进程内存
       │  VirtualQueryEx + ReadProcessMemory
       ▼
  all_keys.json  ─────────────  key_scan.py
       │  AES-256-CBC 解密
       ▼
  decrypted/  ────────────────  decrypt.py
       │
       ├──► chat_list.py ──► chat_export.py ──► exports/*.txt
       │                   ──► employee_match.py ──► employee_export/*.txt
       ├──► wordcloud_gen.py ──► wordcloud/*.html
       └──► report_gen.py ──► report.html
```

### key_scan.py — 密钥提取

**唯一对外入口**：

```python
def run_key_scan(db_dir, out_file, print_fn=None, progress_fn=None) -> dict
```

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `db_dir` | str | 微信 `db_storage` 目录路径 |
| `out_file` | str | 输出 JSON 路径 (如 `output/all_keys.json`) |
| `print_fn` | callable | 日志回调 `fn(str)` |
| `progress_fn` | callable | 进度回调 `fn(pct: int, msg: str)` |

**返回值**：`{salt_hex: enc_key_hex, ...}`

**输出文件格式** (`all_keys.json`)：
```json
{
  "contact/contact.db": {
    "enc_key": "a1b2c3...64hex...",
    "salt": "d4e5f6...32hex...",
    "size_mb": 9.4
  },
  "_db_dir": "D:\\xwechat_files\\wxid_xxx\\db_storage"
}
```

**内部核心函数**：

```python
get_pids() -> list[(pid, mem_kb)]        # 枚举 Weixin.exe 进程
enum_regions(handle) -> list[(addr, sz)] # 枚举可读内存区域
read_mem(handle, addr, sz) -> bytes      # 读取进程内存
verify_enc_key(enc_key, db_page1) -> bool # HMAC-SHA512 验证密钥
collect_db_files(db_dir) -> (db_files, salt_to_dbs)  # 收集 DB 及 salt
scan_memory_for_keys(data, hex_re, ...) -> int       # 正则扫描 + 验证
cross_verify_keys(db_files, salt_to_dbs, key_map, ...) # 交叉验证
```

**算法细节**：
- 正则模式：`x'([0-9a-fA-F]{64,192})'`
- 支持 3 种 hex 长度：64 (纯密钥)、96 (密钥+salt)、>96 (长串含密钥+salt)
- MAC key 派生：`PBKDF2-HMAC-SHA512(enc_key, salt XOR 0x3A, iter=2, dklen=32)`
- HMAC 验证：`HMAC-SHA512(page1[16:4032], mac_key, page_no=1)`

### decrypt.py — 数据库解密

**唯一对外入口**：

```python
def run_decrypt(keys_file, db_dir, out_dir, print_fn=None, progress_fn=None) -> (success, failed, skipped)
```

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `keys_file` | str | `all_keys.json` 路径 |
| `db_dir` | str | 加密 DB 目录 |
| `out_dir` | str | 解密输出目录 |
| `print_fn` | callable | 日志回调 |
| `progress_fn` | callable | 进度回调 `fn(pct, msg)` |

**返回值**：`(success_count, failed_count, skipped_count)`

**内部核心函数**：

```python
derive_mac_key(enc_key, salt) -> bytes         # 派生 HMAC 密钥
decrypt_page(enc_key, page_data, pgno) -> bytes # 单页 AES-CBC 解密
decrypt_database(db_path, out_path, enc_key) -> bool  # 全库解密
```

**SQLCipher 4 参数**：

| 参数 | 值 |
|------|-----|
| 算法 | AES-256-CBC |
| 密钥长度 | 256 bit (32 bytes) |
| 页大小 | 4096 bytes |
| Reserve | 80 bytes (IV 16B + HMAC 64B) |
| KDF | PBKDF2-HMAC-SHA512 |
| KDF 迭代 | 2 |
| 页 HMAC | HMAC-SHA512 |

**每页布局**：
```
Offset  Size  Content
0       16    Salt (仅 page 1 有效)
16      3968  加密数据 (page 1 实际为 3984: 4000-16)
4000    16    IV (AES-CBC 初始向量)
4016    64    HMAC (page 1); page N 则为 4032+64
---
Page 1 特殊处理: 解密后前 16 字节替换为 "SQLite format 3\0"
所有页尾部填充 80 字节零 (reserve)
```

**文件锁定处理**：尝试 `open(db, 'rb')`，失败则 `shutil.copy2` 到 `%TEMP%` 解密。

**完整性验证**：解密后 `sqlite3.connect()` 并读取 `sqlite_master`，最后清理残留 `-wal`/`-shm`。

### employee_match.py — 员工匹配

```python
def run_employee_export(decrypted_dir, excel_path, out_dir,
                        start_ts=None, end_ts=None, keyword=None,
                        min_score=30, name_filter=None, list_only=False,
                        print_fn=None, progress_fn=None) -> (matches, unmatched)
```

**Excel 格式要求**：第一行为表头，自动检测列位置：
- 姓名列：表头含 `名` (不含 `编码`)
- 部门列：表头含 `部门`
- 电话列：表头含 `移动电话`
- 状态列：表头含 `禁用` 或 `状态`
- 大区列：表头含 `大区`

**匹配评分表** (10 级)：

| 分数 | 规则 | 说明 |
|------|------|------|
| 100 | `remark == 员工名` | 备注精确匹配 |
| 90 | `alias == 员工名` | 别名精确匹配 |
| 85 | `员工名 in remark` | 备注包含姓名 |
| 80 | `nick_name == 员工名` | 昵称精确匹配 |
| 70 | `员工名 in nick_name` | 昵称包含姓名 |
| 60 | `description == 员工名` | 描述精确匹配 |
| 50 | `nick_name in 员工名` | 姓名包含昵称 |
| 45 | `remark in 员工名` | 姓名包含备注 |
| 40 | `员工名 in description` | 描述包含姓名 |
| 30 | `description in 员工名` | 姓名包含描述 |

**排除规则**：单字匹配完全跳过（避免大量假阳性）。

|------|------|------|--------|

## 数据库 Schema

### contact.db

```sql
-- 联系人主表
CREATE TABLE contact (
    id INTEGER PRIMARY KEY,
    username TEXT,           -- wxid_xxx 或 xxxxx@chatroom
    local_type INTEGER,
    alias TEXT,              -- 用户设置的别名
    remark TEXT,             -- 备注名
    nick_name TEXT,          -- 用户昵称
    description TEXT,        -- 个性签名/描述
    ...
);

-- 群聊成员关系
CREATE TABLE chatroom_member (
    room_id INTEGER,         -- → contact.id (群聊)
    member_id INTEGER        -- → contact.id (成员)
);

-- 群聊元信息
CREATE TABLE chat_room (
    id INTEGER,
    username TEXT,
    owner TEXT,              -- 群主（可能存群名）
    ext_buffer BLOB
);
```

### session.db

```sql
CREATE TABLE SessionTable (
    username TEXT PRIMARY KEY,
    type INTEGER,
    unread_count INTEGER,
    summary TEXT,            -- "显示名:最后一条消息"
    last_timestamp INTEGER,
    ...
);
```

### message_N.db

```sql
-- 用户名 → rowid 映射
CREATE TABLE Name2Id (
    user_name TEXT PRIMARY KEY,
    is_session INTEGER
);

-- 消息表 (每会话一个, 表名 = 'Msg_' || MD5(username))
CREATE TABLE Msg_<md5hash> (
    local_id INTEGER PRIMARY KEY AUTOINCREMENT,
    server_id INTEGER,
    local_type INTEGER,          -- & 0xFFFFFFFF 得到基础类型
    sort_seq INTEGER,
    real_sender_id INTEGER,      -- → contact.id (发起者)
    create_time INTEGER,         -- Unix 时间戳 (秒)
    status INTEGER,              -- 1=, 2=已发送, 3=已接收, 4=系统
    origin_source INTEGER,       -- 1=自己发出, 2=他人
    source TEXT,                 -- XML 元数据 (群消息存 "sender:\n<xml>")
    message_content TEXT,        -- 文本消息内容 / BLOB protobuf
```

## 密钥与解密算法

### 密钥内存格式

WCDB 在进程内存中缓存格式：
```
x'<64hex_enc_key><32hex_salt>'
```

也可能出现：
- 仅有 enc_key (64 hex) — 需要遍历所有 salt 尝试匹配
- 长串含 enc_key + salt (>96 hex) — 取前 64 + 后 32

### HMAC 验证算法

```
输入: enc_key (32 bytes), db_page1 (4096 bytes)

1. 提取 salt = page1[0:16]
2. 派生 mac_key:
     mac_salt = salt XOR 0x3A (逐字节)
     mac_key = PBKDF2-HMAC-SHA512(enc_key, mac_salt, iter=2, dklen=32)

3. 计算 HMAC:
     hmac_data = page1[16 : 4096-80+16] = page1[16:4032]
     stored_hmac = page1[4096-64 : 4096] = page1[4032:4096]
     computed = HMAC-SHA512(hmac_data, mac_key)
     computed.update(struct.pack("<I", 1))  # page number

4. 验证: computed.digest() == stored_hmac
```

### AES 解密算法

```
参数: AES-256-CBC, key=enc_key(32B), iv=page[PAGE_SZ-80 : PAGE_SZ-64](16B)

Page 1 (特殊):
  加密区 = page[16 : 4096-80]       # 跳过 salt, 不含 reserve
  解密后 = AES_CBC_DEC(加密区)
  输出 = b'SQLite format 3\x00' + 解密后[16:] + zeros(80)

Page N (N >= 2):
  加密区 = page[0 : 4096-80]        # 整页不含 reserve
  解密后 = AES_CBC_DEC(加密区)
  输出 = 解密后 + zeros(80)

说明: reserve=80 包含 IV(16B) + HMAC(64B)
      B-tree 基于 usable_size=4016 (4096-80) 构建
      解密后尾部补零保持页大小不变
```

- 中文 Windows 下 `build.bat` 必须以 UTF-8 with BOM 或纯 ASCII 保存（否则 GBK 乱码）

### 常见问题排查

**1. 密钥提取失败 (0 密钥)**

检查项：
- Weixin.exe 是否正在运行
- 是否有管理员权限（`ReadProcessMemory` 通常不需要，但 VirtualQueryEx 可能需要）
- 微信版本是否为 4.x（3.x 及以下使用不同加密方案）
- 手动验证：用 HxD 等工具在 Weixin.exe 内存中搜索 `x'` 前缀

```python
# 调试：检查找到多少 hex 匹配
# 在 key_scan.py 的 main() 中添加：
print(f"total hex matches: {all_hex_matches}")
print(f"remaining salts: {remaining_salts}")
```

**2. 解密后数据库损坏**

```python
# 手动验证单个 DB
import sqlite3
conn = sqlite3.connect("output/decrypted/contact/contact.db")
print(conn.execute("SELECT COUNT(*) FROM contact").fetchone())
# 预期: (N,) 正常数字
# 异常: sqlite3.DatabaseError: database disk image is malformed
```

常见原因：
- enc_key 错误（HMAC 验证通过了但密钥不对的极端情况）
- 文件被微信中途修改
- 清理 `-wal`/`-shm` 残留文件后重试

**3. 模块导入错误**

打包后运行时，确保所有模块在 `src/` 下使用平级 `import`：
```python
# 正确 (打包后所有 src/ 在同一目录)
from utils import find_wechat_data_dir

# 错误 (打包后没有包结构)
from .utils import find_wechat_data_dir
```

**4. 词云中文乱码**

确保 `jieba` 的字典文件被打包：
```python
# wechat_exp.spec 中必须有:
--collect-all jieba
```

## 已知限制

1. **消息内容编码**：非文本类型 (image/video/voice/emoji) 的 `message_content` 是 Protobuf 二进制，仅有文本类型和 XML 类型可读
2. **图片/文件提取**：`hardlink.db` 中的文件路径引用 `xwechat_files` 下的加密文件名，当前工具不处理图片/文件提取
3. **撤回消息**：被撤回的消息 `message_content` 可能保留 XML 提示文本，但原始内容已从数据库删除
4. **内存扫描时效**：Weixin.exe 重启后部分密钥可能被清除出内存缓存，需重新扫描
5. **非好友发送者**：群聊中非好友的 `real_sender_id` 可能不在 contact 表中，只能通过 Name2Id rowid 有限映射
6. **数据库分片**：message 按大小分片为 `message_0.db`, `message_1.db`...，同一会话可能分布在多个 DB 中，模块已自动处理跨 DB 聚合
7. **仅 Windows**：内存扫描 (ctypes/Win32 API) 和 PyInstaller 打包均仅支持 Windows
8. **微信 4.x 专用**：3.x 及以下版本使用不同的加密方案 (SQLCipher 3, 不同 KDF 参数)

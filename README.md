# WeChat EXP — 便携式微信聊天记录导出分析工具

**单文件、零依赖、复制即用**。在任何 Windows 10/11 电脑上双击 `wechat_exp.exe`，自动检测微信、提取密钥、解密数据库，提供聊天导出、员工批量导出、词云分析、综合报告等功能。

支持 **微信 4.x** (SQLCipher 4, AES-256-CBC)。

## 快速开始

### 用户使用

1. 在**已登录微信**的 Windows 电脑上，双击 `wechat_exp.exe`
2. 看到主菜单后，按 `1` → Enter → 一键完成全流程
3. 导出的聊天记录在 `exp/` 目录

### 开发者构建

```batch
git clone <repo>
cd wechat-exp
pip install -r requirements.txt
python src/main.py          # 直接运行
build.bat                    # 打包为单文件 EXE
```

## 命令行模式

支持命令行参数，无需进入交互菜单，适合脚本/自动化场景：

```batch
# 一键全流程
wechat_exp.exe --auto

# 提取密钥 + 解密
wechat_exp.exe --keys --decrypt

# 按姓名导出最近 7 天
wechat_exp.exe --export --name=张三 --days=7

# 按关键词导出
wechat_exp.exe --export --keyword=报价 --days=30

# 指定微信数据目录
wechat_exp.exe --auto --db-dir=D:\xwechat_files\wxid_xxx\db_storage

# 自定义输出目录
wechat_exp.exe --export --output=C:\Users\Admin\Desktop\chat_export
```

完整参数列表：

| 参数 | 说明 |
|------|------|
| `--auto` | 一键全流程：提取密钥 → 解密 → 导出最近30天 |
| `--keys` | 仅提取密钥 |
| `--decrypt` | 仅解密数据库 |
| `--export` | 导出聊天记录 |
| `--name NAME` | 按姓名过滤 |
| `--days N` | 最近 N 天 |
| `--keyword KW` | 关键词过滤 |
| `--db-dir PATH` | 手动指定微信数据目录 |
| `--output PATH` | 自定义输出目录 |

### 密钥提取原理

微信 4.x 使用 WCDB (SQLCipher 4)，每个数据库文件使用独立的 AES-256 密钥。WCDB 在进程内存中以特定格式缓存密钥：

```
x'<64hex_enc_key><32hex_salt>'
```

提取流程：

1. **枚举进程** — `tasklist` 找到所有 `Weixin.exe` 进程，按内存降序排列
2. **枚举内存区域** — `VirtualQueryEx` 遍历进程地址空间，收集所有已提交的可读区域
3. **正则扫描** — 在每个区域中匹配 `x'([0-9a-fA-F]{64,192})'` 模式
4. **HMAC 验证** — 对候选密钥，读取对应 DB 文件的 page 1，计算 HMAC-SHA512 校验
5. **交叉验证** — 已找到的密钥尝试解密其他 salt 的数据库（多个 DB 可能共享密钥）

```
enc_key ──► PBKDF2-HMAC-SHA512(salt XOR 0x3A, iter=2) ──► mac_key
                                                              │
page1[0:16] = salt                                           │
page1[16:4016] = hmac_data ──────────────────────────────────┤
page1[4032:4096] = stored_hmac ◄── 比较 ──► HMAC-SHA512(hmac_data, mac_key, page_no=1)
```

### 数据库解密原理

SQLCipher 4 参数：`AES-256-CBC`, `page_size=4096`, `reserve=80` (IV 16B + HMAC 64B)。

每页格式：
```
[0..SALT_SZ]        = 16B salt (仅 page 1 用于密钥派生)
[SALT_SZ..PAGE_SZ-RESERVE_SZ] = 加密数据
[PAGE_SZ-RESERVE_SZ..PAGE_SZ-64] = 16B IV
[PAGE_SZ-64..PAGE_SZ] = 64B HMAC
```

解密后还原为标准 SQLite 4096 字节页面，page 1 的前 16 字节写入 `SQLite format 3\0` 文件头。

**文件锁定处理**：如果微信正在运行，DB 文件可能被锁定。此时自动 `shutil.copy2` 到 `%TEMP%`，解密后再写入目标路径，最后清理临时文件。

**完整性验证**：解密后用 `sqlite3.connect()` + 读取 `sqlite_master` 验证 DB 可用，并清理残留的 `-wal`/`-shm` 文件。

### 群聊名称三级解析

微信数据库中群聊以 `xxxxxxxxx@chatroom` 格式存储，无法直接阅读。本工具实现了四级解析链：

```
Level 1 — contact.db 直接匹配
  SELECT COALESCE(remark, nick_name, alias) FROM contact
  WHERE username = '<chatroom_id>'

Level 2 — session.db 摘要提取
  SELECT summary FROM SessionTable WHERE username = '<chatroom_id>'
  → 格式 "显示名:最后一条消息"，提取冒号前部分

Level 3 — chat_room 表 owner 字段
  SELECT owner FROM chat_room WHERE username = '<chatroom_id>'

Level 4 — 降级方案
  → "群聊(<username前12位...>)"
```

### 智能停用词三层过滤

替代传统硬编码停用词表，解决"您好""谢谢""抱拳""的话"等无意义高频词污染：

**第一层 — 静态词表 (~800 词)**
- 语气词：呢/啊/吧/嘛/呀/哦/嗯/哈/呵/哎/喂/哟
- 代词：我/你/他/她/它/我们/你们/他们/这/那/哪/谁/什么
- 连词介词：和/与/或/的/了/在/是/就/把/被/从/对/以
- 礼貌用语：您好/谢谢/不客气/请问/麻烦/拜托
- 微信特定表情文字：抱拳/捂脸/呲牙/玫瑰/爱心/点赞
- 英文停用词：the/a/an/is/are/was/were...

**第二层 — jieba 词性过滤**
- 保留：n* (名词)、v* (动词)、a* (形容词)、eng (英文)、j (简称)、l (习语)、i (成语)
- 剔除：u (助词)、c (连词)、p (介词)、r (代词)、e (叹词)、q (量词)、m (数词)、d (副词)...

**第三层 — 上下文频率降权**
- 词在 > 80% 的消息中出现 → 权重 ×0.3（群通用词如"收到"）
- 词在 > 50% 的消息中出现 → 权重 ×0.6

```python
# 用法示例
from stopwords import is_meaningful, filter_words

keep, weight = is_meaningful("收到", pos="v", context_freq=0.85, total_msgs=1000)
# → (True, 0.3) 保留但降权
```

### 员工匹配引擎

10 级加权评分，每个员工匹配最高分联系人，分数相同则标记歧义：

| 评分 | 匹配规则 | 示例 |
|------|----------|------|
| 100 | 备注完全匹配 | 备注="张三" ← 员工"张三" |
| 90 | 别名完全匹配 | alias="张三" |
| 85 | 备注包含姓名 | 备注="张三(采购部)" ← 员工"张三" |
| 80 | 昵称完全匹配 | nick_name="张三" |
| 70 | 昵称包含姓名 | nick_name="张三-北京" ← 员工"张三" |
| 60 | 描述完全匹配 | description="张三" |
| 50 | 姓名包含昵称 | 员工"张小明" ← nick="小明" |
| 45 | 姓名包含备注 | 员工"李四" ← remark="四" |
| 40 | 描述包含姓名 | description 包含员工名 |
| 30 | 姓名包含描述 | 员工名包含 description |

**单字匹配被排除**：昵称只有 1 个汉字时不再匹配，避免"张"错误匹配到"张三"等大量假阳性。

## 路径检测策略

自动检测微信数据目录的三个策略，按优先级降序：

| 优先级 | 策略 | 说明 |
|--------|------|------|
| 1 | 微信配置解析 | 读 `%APPDATA%\Tencent\xwechat\config\*.ini`，每文件是纯文本路径 |
| 2 | 常见位置 + 定深 glob | `{父目录}/xwechat_files/*/db_storage` (2层，不递归) |
| 3 | 全驱动器遍历 | `GetLogicalDrives()` 获取所有盘符后逐个检查 |

多账号时按 `message` 目录的修改时间自动选择最近活跃的账号，无需用户手动选择。

## 容错与边界情况

| 情况 | 处理 |
|------|------|
| Weixin.exe 未运行 | 提示"请先启动微信"，等待后重试 |
| 部分密钥未找到 | 输出 MISSING 列表，尝试交叉验证 |
| DB 文件被微信锁定 | 自动 `shutil.copy2` 到 `%TEMP%` 解密 |
| 解密后 DB 损坏 | `PRAGMA integrity_check` 等效验证，失败标记 SKIP |
| 联系人无聊天记录 | 跳过，不生成空文件 |
| Excel 未找到 | 提示"请将员工信息.xlsx 放在同目录" |
| 词云无有意义词 | 降低过滤阈值，提示内容较少 |
| 新版本微信变更 | 正则 `x'([0-9a-fA-F]{64,192})'` 足够宽泛 |
| 磁盘空间不足 | 解密前估算空间，不足时警告 |

## FAQ

**Q: 提示"Weixin.exe 未运行"怎么办？**
A: 先启动微信并登录，再运行本工具。微信必须保持运行状态才能提取密钥。

**Q: 部分数据库"无密钥"？**
A: 尝试在微信中打开对应聊天窗口，然后重新提取密钥。微信会按需加载密钥到内存。

**Q: 员工匹配结果太少？**
A: 检查微信联系人的备注名是否包含员工姓名。在微信中给同事设置备注名可以显著提高匹配率。也可降低 `--min-score` 参数。

**Q: 词云中仍有无意义词汇？**
A: 可以在 `src/stopwords.py` 的 `STOP_WORDS` 集合中添加自定义停用词，重新打包即可。

**Q: 导出的文件在哪里？**
A: 聊天导出在 `exp/` 目录，其他输出在 `output/` 目录。程序运行完成后会显示完整路径。

**Q: 支持 macOS/Linux 吗？**
A: 本次仅支持 Windows。macOS/Linux 微信的密钥存储机制不同。

**Q: 微信更新后还能用吗？**
A: 只要微信继续使用 WCDB (SQLCipher) 且 `x'...'` 格式的密钥缓存不变，核心机制就继续有效。
